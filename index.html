Krug Programming Language

Krug is a programming language for systems programming. The compiler is a compiler-as-a-service, which means that the compilers runs as a server. Each compilation phase is cut up into an API route which can be invoked from a client.

Try Krug

The language is a work in progress. Nothing works yet, and the language/compiler is far from stable. You can still try and get a krug program to compile.

Using the JS frontend

Krug offers a few choices in frontend, the standard frontend being 'krug', a frontend that is implemented in JavaScript.

JavaScript was chosen to be the main frontend due to its availability. You will need a few things to get started:

  • yarn (or npm)
  • node
  • krug-serv

First, ensure that an instance of 'krug-serv' is running locally on your machine. It should be running, by default, on port 8001. Take a look at the krug-serv repository to find out more. 

yarn add global krug
krug hello_world.krug

krug-go

Additionally, there is a frontend that is written in Go. This was the initial frontend, though krug was later made the standard frontend due to its easy adoption. If you want to use the Go version of the krug frontend, there are further instructions on the krug repository, though it is not as actively maintained as krug at this time.

Goals

There are many goals for this project, here are some that are not so obvious:

  • Setup a server to run a compiler instance for demoing the frontend(s). This would be running at 'serv.krug-lang.org'.
  • Have a compiler frontend written in Krug very early on in the development of the compiler.
  • Potentially support LLVM as the (core) code generation framework for the compiler.
